# 过拟合检测以及防止过拟合方法与案例

本文档用于记录本轮“放量长上影”策略研究里形成的过拟合检测方法、判断标准、处理手段，以及对应的真实案例。

核心观点很简单：过拟合不能靠一句“应该没问题”来排除，只能靠一条完整证据链来降低风险。一个策略从形态事件、过滤条件、排序因子、组合方式到真实撮合执行，每一层都可能把原本稳定的信号放大成脆弱的参数尖峰。

## 零、本项目当前落地口径

本文档不是只用于解释概念，而是作为本项目判断“研究候选 / 成熟候选 / 成熟策略”的执行标准。

当前 RPS 策略升级为成熟策略，必须同时满足以下硬门槛：

1. 收益和 Calmar 保真。  
   不能用低收益、低 Calmar 的防守线冒充高收益策略通过。替换策略必须和原目标属于同一收益/回撤级别，否则只能标记为低波动观察线。

2. 三段表现不能断裂。  
   训练集、测试集、验证集都要为正，且不能出现测试集或验证集明显失效。

3. 关键参数不是孤岛。  
   邻域中位数 / 当前点不应低于 `70%`，当前点 / 邻域中位数不应高于 `1.5`。如果当前点超过邻域中位数 `2` 倍，基本视为参数尖峰。

4. Top20 利润占比不能超过 `100%`。  
   超过 `100%` 说明除前 20 笔最大盈利交易以外，其余交易整体亏损，策略收益高度依赖少数历史右尾样本。

5. 真实撮合必须通过。  
   A 股短线策略必须纳入 T+1、涨跌停、停牌、手续费、滑点、100 股整手等约束。

6. clean-forward 不少于 20 个交易日。  
   冻结规则之后的真实前推少于 20 个交易日，只能说明“暂时没坏”，不能证明策略已经通过干净前推。前推期间不能继续按结果反向调参。

7. 研究记录必须可复现。  
   策略名称、参数、切分区间、命令、输出路径、失败原因和升级原因都要落盘。

当前项目里，`Top20 利润占比` 的正常参考区间是：

| Top20 利润占比 | 判断 |
| ---: | --- |
| `< 50%` | 收益来源非常分散 |
| `50% ~ 70%` | 健康常见区间 |
| `70% ~ 90%` | 偏集中，但高弹性短线策略仍可接受 |
| `90% ~ 100%` | 临界区，必须配合 remove-top 压力测试 |
| `> 100%` | 高风险，不应直接升级为成熟策略 |

所以，`119.65%` 不是“利润很高”的意思，而是“前 20 笔赢家贡献超过全部净利润，其余交易合计拖后腿”。这种结构不能直接放进成熟池。

## 一、什么是策略过拟合

在量化策略里，过拟合通常不是代码写错，而是研究过程把历史样本中的偶然性当成了规律。

常见形式包括：

1. 参数过拟合  
   某个参数点表现极好，但左右微调后收益、回撤、卡玛快速恶化。例如阈值从 `0.60` 改到 `0.59` 或 `0.61` 后，最终组合卡玛接近腰斩。

2. 条件堆叠过拟合  
   每个过滤条件单看都有一点解释力，但叠加后样本变得很少，最终只是在筛历史上刚好涨过的一小批股票。

3. 排序层过拟合  
   母池本身稳定，但排序因子、选股数量、持仓周期、offset 组合之后，某个参数突然变成尖峰。这种情况更隐蔽，因为看基础事件时问题不明显。

4. 组合层过拟合  
   单腿策略各自看起来可以，组合后因为相关性、仓位叠加、回撤错位等原因，最终表现依赖某几个历史窗口。

5. 执行层过拟合  
   本地快速框架和 PRO 框架撮合假设不一致，例如是否涨停不买、涨停不卖、跌停锁仓、开盘成交、T+1 约束等，会导致回测结果差距很大。

6. 样本切分过拟合  
   开发过程中反复看全样本，最后的所谓测试集已经被研究过程污染。尤其是频繁用 2026 年数据调参数时，2026 年就不能再被当作完全干净的验证集。

## 二、过拟合的典型信号

以下情况出现越多，过拟合风险越高：

- 参数只在一个精确点有效，例如 `0.60`、`4.0`、`0.75` 这种点位附近没有平原。
- 参数向左或向右轻微移动，年化、回撤、卡玛明显恶化。
- 单个过滤条件的业务解释弱，但对结果提升极大。
- 策略收益主要来自极少数年份、极少数月份、极少数交易日。
- 平均持股数量明显下降，但收益上升，可能是在用过滤条件挑历史幸运样本。
- 样本内强、样本外弱，或者 2026 年单独强但历史其它年份一般。
- 本地框架很好，PRO 框架差很多，且差异不能用执行规则解释清楚。
- 快速模式很好，真实撮合模式差很多。
- 参数微调后信号重合度低，说明策略选出的股票不是同一类机会。
- 过滤条件越来越多，但每个新增条件的逻辑解释越来越弱。

## 三、检测过拟合的基本流程

### 1. 先固定研究环境

每次判断一个策略是否可靠之前，先固定以下内容：

- 数据版本。
- 复权方式。
- 股票池。
- 是否剔除 ST、退市、北交所、科创板等。
- 买卖时间，例如 `close-open`。
- 是否涨停不买。
- 是否涨停不卖。
- 是否跌停锁仓。
- 是否遵守 A 股 T+1。
- 手续费、滑点、冲击成本。
- 本地框架和 PRO 框架是否用同一套执行规则。

如果这些条件没固定，后面所有收益对比都可能是无效对比。

### 2. 拆母池，不要直接看最终策略

一个形态策略至少要拆成这些层次：

1. 纯形态母池  
   只看最原始的形态定义，例如放量、上影线、成交额阈值。

2. 公共基础池  
   加入最核心、最稳定、最有业务解释的过滤条件。

3. 原始论文或原始想法池  
   如果策略来自某篇 PDF、论坛思路或旧版本策略，要保留它的原始定义，避免后续改造后说不清来源。

4. 排序选股层  
   加入排序因子、选股数量、offset、持仓周期。

5. 单腿策略层  
   每条腿独立评估收益、回撤、卡玛、信号数量、持股数量。

6. 组合层  
   多腿等权、加权、去重、仓位分配后的最终效果。

过拟合检测不能只看最终组合。最终组合很容易把多个小噪声叠加成漂亮曲线。

### 3. 做单参数邻域扫描

对每个关键参数，至少检查：

- 当前参数点。
- 左右近邻。
- 更宽一点的参数区间。
- 参数收益曲线是否平滑。
- 最优点是不是孤立尖峰。

例如阈值 `上影线比例 >= 0.60`，不能只看 `0.60`，还要看：

- `0.58`
- `0.59`
- `0.60`
- `0.61`
- `0.62`

如果只有 `0.60` 特别好，其它都明显差，就是高风险。

### 4. 做多参数邻域扫描

单参数稳定不代表组合稳定。常见情况是：

- A 参数单独看稳定。
- B 参数单独看稳定。
- A 和 B 同时变化后不稳定。

因此要看局部二维或多维邻域，例如：

- `上影线比例` × `成交额比Mean`
- `上影线比例` × `收盘N日高点比`
- `成交额比Mean` × `选股数量`
- `选股数量` × `持仓周期`

判断时不要只找最优点，而要看最优点周围是不是一片相对平坦的区域。

### 5. 做训练、测试、验证切分

推荐至少切成三段：

- 训练集：用于找想法、调参数、筛条件。
- 测试集：用于判断策略是否从训练集泛化。
- 验证集：尽量保留到最后，只用于最终确认。

本轮研究里曾使用 `2026 年至今` 作为验证集。这种做法可以用于阶段性检查，但如果后续反复根据 2026 年结果调参数，2026 年就不再是干净验证集。

更严格的做法是：

- 开发阶段只看训练集和测试集。
- 2026 年只在策略冻结后看一次。
- 如果看完 2026 年又改策略，需要重新标记它只是“候选策略”，不能直接升级为成熟策略。

### 6. 看逐年表现和市场状态

不要只看总年化和总卡玛。至少要看：

- 每年收益。
- 每年最大回撤。
- 每年卡玛。
- 牛市、熊市、震荡市分别表现。
- 极端行情期间表现。
- 注册制前后是否稳定。
- 量化拥挤或微盘风格变化阶段是否稳定。

如果策略只靠某一年贡献大部分收益，风险很高。

### 7. 看信号数量和持仓数量

过滤条件带来的提升，要和样本数量变化一起看。

重点看：

- 总信号数。
- 每年信号数。
- 每日平均持股数。
- 空仓天数。
- 参数变化后信号保留率。
- 参数变化后信号重合度。

一个过滤条件如果只过滤掉很少信号，却稳定提升收益，通常更可信。  
如果过滤掉大量信号，收益大幅提升，就要怀疑它是不是在挑历史幸运样本。

### 8. 看信号重合度

参数微调后，不只看收益，也要看选出来的股票是不是同一批。

可用指标：

- Jaccard 重合度。
- 原始信号保留率。
- 新增信号占比。
- 每年重合度。
- 亏损期重合度。

如果 `0.60` 和 `0.59` 选出来的股票差异很大，即使两个参数收益都不错，也说明策略对阈值比较敏感。

### 9. 最后用真实执行规则复核

快速框架可以用于大规模搜索，但成熟策略必须用真实执行规则复核。

至少要复核：

- A 股 T+1，不能出现 T+0 逻辑。
- 开盘买入是否遇到涨停买不进。
- 卖出时是否涨停不卖。
- 卖出时是否跌停锁仓。
- 停牌如何处理。
- 一字板如何处理。
- 资金分配和仓位上限。
- 手续费和滑点。

如果快速模式和真实模式差距很大，要先解释差距来源，再谈策略是否有效。

## 四、建议使用的稳定性指标

### 1. 邻域收益保留率

定义：

```text
邻域收益保留率 = 邻近参数表现 / 当前参数表现
```

可用于年化、卡玛、收益回撤比等指标。

参考判断：

- `>= 85%`：相对稳定。
- `70% ~ 85%`：需要谨慎。
- `< 70%`：高过拟合风险。
- `< 50%`：基本可以认为是参数尖峰。

### 2. 邻域中位数比值

定义：

```text
邻域中位数比值 = 附近参数表现中位数 / 当前参数表现
```

这个指标比单看左右两个点更稳。  
如果当前参数远高于邻域中位数，说明当前点可能是孤岛。

参考判断：

- `>= 85%`：参数附近有平原。
- `70% ~ 85%`：略敏感。
- `< 70%`：明显敏感。
- `< 50%`：强烈怀疑过拟合。

### 3. 当前点相对邻域倍数

定义：

```text
当前点相对邻域倍数 = 当前参数表现 / 邻域中位数表现
```

参考判断：

- `< 1.2`：较稳定。
- `1.2 ~ 1.5`：可接受但要继续观察。
- `1.5 ~ 2.0`：有明显优化痕迹。
- `> 2.0`：高风险。

### 4. 训练、测试、验证一致性

不能只看全样本。更可靠的情况是：

- 训练集有效。
- 测试集有效。
- 验证集没有明显失效。
- 三段的回撤都可接受。
- 三段收益来源逻辑一致。

危险情况：

- 训练集很好，测试集明显变差。
- 测试集很好，训练集一般，可能是后验挑出来的。
- 2026 年特别好，但历史长期一般，可能是近期行情拟合。

### 5. 信号数量稳定性

参考判断：

- 参数微调后，平均持股数量变化很小，收益也接近，较稳定。
- 参数微调后，持股数量变化很小，但收益差很多，要警惕排序或个股样本噪声。
- 参数微调后，持股数量变化很大，收益也差很多，说明条件本身过强。

### 6. 头部利润集中度

如果策略总收益主要靠少数大赢家贡献，就算年化和卡玛很好，也要警惕右尾依赖。

常用指标：

```text
TopN 利润占比 = 前 N 笔盈利交易的利润合计 / 全部交易净利润
```

本项目默认使用 `Top20 利润占比`。如果该指标超过 `100%`，说明除前 20 笔最大盈利交易以外，其余交易合计为亏损，策略收益结构高度依赖少数历史样本。

参考判断：

- `< 50%`：收益来源比较分散。
- `50% ~ 70%`：正常。
- `70% ~ 90%`：偏集中，但高弹性短线策略可以接受。
- `90% ~ 100%`：临界，需要配合 remove-top 压力测试。
- `> 100%`：高风险，不应直接升级为成熟策略。

配套压力测试：

```text
remove_top1 / remove_top3 / remove_top5 / remove_top10
```

如果移除少数最大盈利交易后，年化、测试集表现或回撤明显恶化，说明策略可能是在捕捉历史右尾幸运样本，而不是稳定规律。

## 五、防止过拟合的方法

### 1. 参数要有业务含义

参数不是不能调，但要能解释。

例如：

- `收盘价 > 开盘价`：表示炸板后仍然收阳，承接不弱。
- `收盘价 > MA20`：表示仍在中期趋势上方。
- `20日收益率Std <= 0.05`：排除波动过大的纯情绪票。
- `成交额比Mean <= 4.0`：排除放量过度、筹码交换过激的票。

如果一个参数只是因为历史回测最好，但说不出市场含义，就不能直接作为成熟参数。

### 2. 优先选择参数平原，不选择最高点

策略开发不是找历史最高收益，而是找稳定区域。

正确思路：

- 最高点卡玛 5.0，但附近都是 2.0，不如不用。
- 某片区域卡玛都在 4.0 左右，哪怕最高只有 4.3，也更可信。

成熟策略宁愿牺牲一点年化，也要降低参数孤岛风险。

### 3. 用参数集成代替单点参数

当某个阈值有解释力但单点过于敏感时，可以用邻域参数集成。

例如本轮放量长上影中，单点 `上影线比例 >= 0.60` 在最终组合层明显敏感。处理方式不是继续押注 `0.60`，而是用：

```text
0.59 / 0.60 / 0.61 = 1 / 2 / 1
```

这样做的含义是：

- 保留 `0.60` 作为中心。
- 让 `0.59` 和 `0.61` 分担阈值误差。
- 降低单个参数点失效对组合的影响。

### 4. 能用排序就少用硬过滤

硬过滤会突然切断样本，容易制造参数尖峰。  
如果某个变量是连续强弱关系，优先考虑作为排序因子，而不是用一个精确阈值过滤。

例如：

- 如果“越接近高点越好”是连续关系，可以考虑排序。
- 如果只在 `0.75` 这个点附近有效，而 `0.70` 和 `0.80` 都不行，就要警惕。

### 5. 控制过滤条件数量

过滤条件越多，越容易从历史中筛出幸运样本。

推荐原则：

- 公共基础条件保留强逻辑条件。
- 弱逻辑条件必须通过稳定性测试。
- 每新增一个条件，都要记录它带来的增益、过滤信号数量、平均持股变化。
- 如果两个条件表达的市场含义接近，尽量只留一个。

### 6. 不要边看验证集边改策略

验证集的价值在于“没被看过”。  
如果看完验证集后又根据它修改参数，它就变成训练集的一部分。

更合理的流程：

1. 在训练集提出策略。
2. 在测试集筛掉明显无效策略。
3. 冻结配置。
4. 看验证集。
5. 如果验证集不满意，回到候选状态，重新开始一轮研究。

### 7. 成熟策略要保留研究记录

不能只保留最终 config。至少要记录：

- 策略来源。
- 开发路径。
- 参数扫描结果。
- 样本内外表现。
- 真实执行规则。
- 最终配置。
- 为什么标记为成熟策略。
- 哪些风险还没完全排除。

这样以后回头看策略时，不会只记得“年化一百多”，却忘了当时是怎么得到的。

## 六、案例：放量长上影策略过拟合检测

### 1. 策略背景

本轮研究的核心形态是“放量长上影”。

基础含义：

- 当天有明显放量。
- 当天冲高回落，留下较长上影线。
- 但收盘不能太弱，最好仍然收阳或站在中期均线上方。
- 目标是寻找炸板、冲高失败后仍有资金承接的短线反包或修复机会。

后续研究中，策略逐步加入了：

- `收盘价 > 开盘价`
- `收盘价 > MA20`
- `20日收益率Std <= 0.05`
- `收盘均线比 <= 1.15`
- `成交额比Mean <= 4.0`
- `收盘20日涨跌幅 0 ~ 0.60`
- `收盘N日高点比 >= 0.75`
- 排序因子：收盘位置、资金流成交额占比等。

其中部分过滤条件被证明影响不大，部分条件存在参数敏感风险。

### 2. 本轮重点问题

最核心的问题是：

```text
上影线比例 >= 0.60
```

这个参数是否可靠？

如果 `0.60` 附近都有不错表现，说明它可能是稳定形态阈值。  
如果只有 `0.60` 好，`0.59` 和 `0.61` 明显变差，说明它可能是过拟合参数。

### 3. 分层检测结果

本轮没有只看最终策略，而是按开发路径分层检查：

| 层级 | 检测对象 | 结论 |
| --- | --- | --- |
| 纯形态母池 | 只看放量和上影线等原始形态 | `0.60` 不是明显孤岛 |
| 当前公共基础池 | 形态 + 收盘价大于开盘价 + 收盘价大于 MA20 | `0.60` 仍不是明显孤岛 |


### 7. 本轮过拟合诊断结论

对放量长上影策略的诊断是：

1. 形态母池不是主要问题。  
   纯形态层和公共基础池里，`上影线比例 0.60` 附近表现比较平滑。

2. 主要问题出现在排序和组合之后。  
   加入排序因子、选股数量、单腿构建、多腿组合后，`0.60` 被放大成明显尖峰。

3. 最终组合的单点版本有过拟合嫌疑。  
   真实执行模式下，`0.59` 和 `0.61` 都明显弱于 `0.60`。

4. 策略不能简单废掉。  
   因为底层形态不是完全无效，问题是后续构造方式对参数太敏感。

5. 更合理的处理方式是参数集成。  
   用邻域阈值组合降低单点依赖。

## 七、案例：抗过拟合上影集成方案

### 1. 改造目标

原始单点版本的问题是：

```text

```

改造目标不是继续追求最高年化，而是：

- 降低参数孤岛风险。
- 保留核心形态逻辑。
- 降低最大回撤。
- 保持样本内外都有效。
- 使用真实执行规则复核。

### 2. 改造方法

采用上影线比例邻域集成：

```text

```

同时调整高点约束：

```text

```

集成含义：

- `0.60` 仍是中心参数。
- `0.59` 和 `0.61` 作为邻域保护。
- 如果未来真实市场中最优阈值轻微漂移，策略不至于完全失效。

## 八、成熟策略准入清单

一个策略要标记为成熟策略，建议至少满足以下条件：

1. 策略逻辑清楚。  
   能说清楚赚的是什么钱，不只是回测曲线好。

2. 母池有效。  
   原始事件层有正向收益或明确的风险收益特征。

3. 关键参数不是孤岛。  
   邻域表现不能大幅崩掉。

4. 过滤条件有解释。  
   每个过滤条件都要能解释为什么保留。

5. 新增过滤条件有增益记录。  
   要知道它过滤了多少信号，提升了多少收益，改变了多少持仓。

6. 样本内外一致。  
   训练集、测试集、验证集不能严重背离。

7. 逐年表现可接受。  
   不能只靠某一年撑起整体曲线。

8. 真实执行通过。  
   必须遵守 A 股 T+1、涨跌停、停牌、手续费等约束。

9. Top20 利润占比不过红线。  
   默认不能超过 `100%`；`90% ~ 100%` 要继续做 remove-top 压力测试。

10. clean-forward 足够。  
   冻结后真实前推至少积累 `20` 个交易日，少于这个长度只能作为观察结果。

11. 收益和 Calmar 不降级。  
   如果目标是高收益/高 Calmar 策略，不能用低收益防守线凑“通过数量”。

12. 本地框架和 PRO 框架能解释差异。  
   如果两个框架差很多，必须先查清楚原因。

13. 有完整落盘记录。  
   包括 config、结果、参数扫描、结论、风险说明。

## 九、成熟策略和候选策略的区别

### 成熟策略

可以进入常规跟踪或小资金试运行，但仍需定期复核。

要求：

- 逻辑完整。
- 参数稳定。
- 真实撮合通过。
- 样本内外表现一致。
- 研究记录完整。

### 成熟候选

看起来有价值，但还存在明显风险。

常见原因：

- 参数有一定敏感性。
- 验证集看过多次。
- 某个年份贡献过大。
- 信号数量偏少。
- PRO 和本地结果尚未完全对齐。

### 研究候选

只是一个想法或初步结果。

常见原因：

- 只跑了全样本。
- 没有参数扫描。
- 没有真实执行。
- 没有样本外。
- 逻辑解释还不充分。

## 十、过拟合红线

以下情况不建议标记为成熟策略：

- 只有一个参数点能赚钱。
- 左右邻域卡玛低于当前点的 50%。
- 当前点相对邻域中位数超过 2 倍。
- 过滤条件没有业务解释。
- 过滤条件越来越多，但不知道每个条件贡献什么。
- 用 T+0 逻辑回测 A 股短线策略。
- 没有涨跌停约束。
- 只看本地快速框架，不做真实撮合复核。
- 看完验证集后继续调参数，还把验证结果当样本外。
- 结果和 PRO 框架差很多，但没有解释。

## 十二、本轮放量长上影给出的经验

1. 高年化不等于低过拟合。  
   原始单点版本全样本年化很高，但 `0.59` 和 `0.61` 明显变弱。

2. 母池稳定不代表最终策略稳定。  
   本轮 `0.60` 在母池层是平滑的，但在排序和组合层变成尖峰。

3. 参数孤岛可能是后处理制造的。  
   排序因子、选股数量、offset、组合方式都可能放大样本噪声。

4. 抗过拟合通常要牺牲收益。  
   集成版本年化低于单点版本，但回撤更小，卡玛更稳。

5. 策略开发要保留路径。  
   只保留最终 config，后面很难判断参数是逻辑得来的，还是历史拟合出来的。

6. A 股短线策略必须重视执行。  
   T+1、涨跌停、停牌、开盘成交约束都会显著影响结果。

7. 成熟策略不是一次回测决定的。  
   成熟策略需要逻辑、参数、样本外、执行、记录同时过关。
